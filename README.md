# 잇집 프론트엔드

### commit convention
✨ feat: 기능 추가, 삭제, 변경

🐛 fix: 버그, 오류 수정

📝 docs: readme.md, json 파일 등 수정, 라이브러리 설치 (문서 관련, 코드 수정 없음)

💄 style: CSS 등 사용자 UI 디자인 변경 (제품 코드 수정 발생, 코드 형식, 정렬, 주석 등의 변경)

♻ refactor: 코드 리팩토링

🧪 test: 테스트 코드 추가, 삭제, 변경 등 (코드 수정 없음, 테스트 코드에 관련된 모든 변경에 해당)

🔧 config: npm 모듈 설치 등

🌱 chore: (코드의 수정 없이) 설정 변경
